      Surfaces

      Where the brain shows up

      Every surface talks to the same governed brain over MCP: a coding assistant in the IDE, a grounded chat assistant, this app, or one you build. Point a new client at the endpoint and it works immediately, with no copy of the data and no new access path to secure.

      Every surface authenticates the same way. A caller signs in with Google, remote MCP clients through the OAuth flow and scripts with a Google-signed token; the brain verifies it, resolves the principal against the policy, and clamps every read and write to the domains they may see, so one endpoint safely serves an IDE, an assistant and this UI at once.

      Connect an agent over MCP

      Hyper Brain is an MCP server over streamable HTTPS with an in-tenancy OAuth 2.1 server in front. An assistant like Claude connects by signing in with Google, then reads the brain and proposes notes, clamped to the domains that person may see.

      Add to Claude (recommended, no token)
      1. In Claude, open Settings → Connectors → Add custom connector.
      2. Paste the endpoint URL below as the connector URL, and click Add.
      3. Click Connect, then sign in with Google when the browser opens.
      4. Done. Claude can now search, answer and propose, scoped to your access.
      Endpointhttps://hyper-brain-xxxxxx-nw.a.run.app/mcp

      There is no token to paste and no app to register: the brain advertises its OAuth server (protected-resource metadata), the client registers itself, and Google signs you in with PKCE. Works the same in any MCP client that supports custom connectors (Claude web and desktop, ChatGPT).

      For scripts & machine clients (token auth) 
      {
  "mcpServers": {
    "hyper-brain": {
      "type": "http",
      "url": "https://hyper-brain-xxxxxx-nw.a.run.app/mcp",
      "headers": {
        "Authorization": "Bearer ${HYPER_BRAIN_TOKEN}"
      }
    }
  }
}

      For a non-interactive client, mint a token with gcloud auth print-identity-token (audience = the service URL) or impersonate a service account. Every call is verified and domain-scoped by policy.

      Conversations · Microsoft Teams

      Bring a Teams chat or channel thread into the brain as a governed, provenance-stamped note, cleaned into a readable, speaker-attributed conversation.

      Import one now (demo)
      1. Export the messages as JSON from the Graph Explorer, e.g. GET /chats/{id}/messages or /teams/{id}/channels/{id}/messages.
      2. In Studio, choose the Chat / transcript source.
      3. Paste the JSON (or upload the .json), generate the draft, then create or propose it.

      A plain copy-paste of a thread works too; the importer detects Teams/Graph JSON and falls back to plain text.

      Wire a live connector (your tenant)
      1. Register an app in Microsoft Entra ID (Azure AD) in your tenant.
      2. Grant Microsoft Graph application permissions Chat.Read.All / ChannelMessage.Read.All and get admin consent.
      3. Point a scheduled ingest at the Graph messages endpoint, landing into a chosen domain, through the same review as every other source.

      The live connector needs your own Entra app and admin consent, so it is a one-time configuration in your tenancy; the import above needs none of that.

      Voice & meetings

      Turn a meeting recording's transcript into a note: the tacit knowledge only ever said out loud, made searchable and citable like everything else.

      Import one now (demo)
      1. Download the transcript from your meeting tool: a .vtt, .srt or plain text file.
      2. In Studio, choose the Chat / transcript source, then upload the file or paste the transcript.
      3. Timestamps and cue numbers are stripped and speakers kept; generate the draft, then create or propose it.
      Wire a live connector (your platform)
      1. Enable transcription/recording in your meeting platform so a transcript is produced per call.
      2. Have the platform deliver the transcript file (webhook or export) to an ingest bucket.
      3. A scheduled ingest parses and lands it into a chosen domain, through the same review as every other source.

      Formats supported by the importer today: WebVTT (.vtt), SubRip (.srt) and plain text.
