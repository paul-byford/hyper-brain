      Sources

      Onboarding content

      Connectors bring knowledge into the brain: files, web and code today, and conversations, meeting transcripts and even autonomous agents through the same governed pipeline. Agents are the neat case: they sit on both sides, drafting new notes and drawing on the result. Onboarding a source is a configuration, not a rebuild.

      Surfaces

      Where the brain shows up

      Every surface talks to the same governed brain over MCP: a coding assistant in the IDE, a grounded chat or voice assistant, this app, or one you build. Point a new client at the endpoint and it works immediately, with no copy of the data and no new access path to secure.

      Every surface authenticates the same way. A caller presents a Google-signed OIDC token; the brain verifies it, resolves the principal against the policy, and clamps every read and write to the domains they may see, so one endpoint safely serves an IDE, an assistant and this UI at once.

      Built for trust, scale and reach

      Reference architecture

      Hyper Brain on Google Cloud

      Every box is a resource in one Google Cloud project. Compute is scale-to-zero Cloud Run; corpus, embeddings and answer synthesis never leave the tenancy. Only three things sit outside it: the clients that call in, Google sign-in, and the content sources you ingest.

      MCP connector

      Hyper Brain is an MCP server over streamable HTTPS. Point any MCP-capable client at the endpoint, authenticate with a Google-signed token, and it can read the brain, clamped to the domains that caller may see.

      Endpointhttps://hyper-brain-xxxxxx-nw.a.run.app/mcp
      AuthAuthorization: Bearer <Google OIDC token>
      Client config 
      {
  "mcpServers": {
    "hyper-brain": {
      "type": "http",
      "url": "https://hyper-brain-xxxxxx-nw.a.run.app/mcp",
      "headers": {
        "Authorization": "Bearer ${HYPER_BRAIN_TOKEN}"
      }
    }
  }
}

      Remote connectors (Claude / ChatGPT): just add the endpoint URL as a custom connector and sign in with Google, with no token to paste. For scripts and machine-to-machine clients, mint a token with gcloud auth print-identity-token (audience = the service URL) or impersonate a service account. Every call is verified and domain-scoped by policy.
