# Hyper Brain UI

The company's memory, made tangible (ARCHITECTURE.md section 9): a two-page,
dependency-free single-page app that holds no secrets and enforces nothing. It
renders the exported index and policy; the trust boundary stays in the server
(section 7). In a live deployment the same data comes from the IAM-gated,
ACL-enforcing brain.

![Hyper Brain, the Connections page: an animated Sources to Hyper Brain to Surfaces flow, with source categories and the onboarding pipeline](../docs/connections.png)

## Two pages

- **Connections** is how knowledge gets in and where it goes out. An animated
  *Sources → Hyper Brain → Surfaces* flow, the six source categories (files, web,
  code, conversations, voice & meetings, agents), the onboarding pipeline
  (Fetch → Parse → Curate → Land → Index), the surfaces (coding assistants, the
  grounded assistant, this app, embedded/API) with a live MCP connector, and the
  architecture behind it.
- **Explore** is the knowledge itself: a canvas force-graph of the wikilink network
  coloured by domain, a domain browser and reading view, a search/answer panel with
  citations and an honest gap statement, and an "acting as" identity switch where
  **isolation is a visible feature**: switching identity re-lights the sub-graph
  that caller may read. Explore / Split / Read modes trade space between the graph
  and the reading column.

## Run it

```sh
./brain ui           # exports data + serves at http://localhost:8000  (Windows: .\brain.ps1 ui)
```

Or manually:

```sh
python scripts/export_ui_data.py       # writes ui/data/{index,policy,config}.json
python -m http.server 8000 --directory ui
```

Then open <http://localhost:8000/>. On **Explore**, switch the identity dropdown
between `brain-admins` (both domains), `finserv-eng` and `recruiting`, and watch the
graph, document list and search shrink to that identity's permitted domains.

To show the real MCP endpoint in the Connections connector modal, pass it when
exporting:

```sh
python scripts/export_ui_data.py --mcp-url https://<brain-service-url>/mcp
```

## Files

- `index.html` / `styles.css`: the two-page shell and the black-and-gold editorial
  theme (self-hosted Inter Tight + JetBrains Mono, squared corners, light + dark).
- `lib.js`: pure logic (domain ACL, scoped search, answer, graph), no DOM; unit-tested.
- `app.js`: DOM wiring, the canvas knowledge graph and the animated flow diagram.
- `fonts/`: self-hosted woff2 (latin subset). `favicon.svg`: the HB mark.
- `test/lib.test.mjs`: node logic tests (isolation, scoped search, filtering).
- `data/`: generated by the exporter (git-ignored): `index.json`, `policy.json`, `config.json`.

Domains are shown with a categorical colour palette **and named** on every node,
legend and chip, so identity is never colour-alone.
