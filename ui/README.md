# Brain Explorer

A dependency-free single-page app that makes the knowledge tangible
(ARCHITECTURE.md section 9): a wikilink graph coloured by domain, a domain browser
and markdown viewer, a search/answer panel with citations and an honest gap
statement, and a "who am I and what can I see" panel where **isolation is a visible
feature** rather than an invisible check.

The UI holds no secrets and enforces nothing. It renders the exported index and
policy; switching identity re-computes the allowed domains and filters the graph,
browser and search to the sub-graph that caller may read. In a live deployment the
same data comes from the IAM-gated, ACL-enforcing brain (the trust boundary stays
in the server, section 7).

## Run it

```sh
./brain ui           # exports data + serves at http://localhost:8000  (Windows: .\brain.ps1 ui)
```

Or manually:

```sh
python scripts/export_ui_data.py       # writes ui/data/{index.json,policy.json}
python -m http.server 8000 --directory ui
```

Then open <http://localhost:8000/>. Try switching the identity dropdown between
`brain-admins` (both domains), `finserv-eng` and `recruiting`, and watch the graph,
document list and search results shrink to that identity's permitted domains.

## Files

- `index.html` / `styles.css` - the shell and the (validated data-viz) palette.
- `lib.js` - pure logic (domain ACL, scoped search, answer, graph), no DOM.
- `app.js` - DOM wiring and the SVG force-graph.
- `test/lib.test.mjs` - node logic tests (isolation, scoped search, filtering).
- `data/` - generated by the exporter (git-ignored).

Colours use the two-slot categorical palette from the `dataviz` skill (blue +
aqua), validated for colour-vision safety; the domain is also named on every node,
legend and chip, so identity is never colour-alone.
