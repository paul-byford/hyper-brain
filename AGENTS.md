# Notes for AI agents working in this repo

This is a **public** repository. Never commit personal or deployment-specific
details into tracked files (code, tests, docs, Terraform, READMEs, UI).

## Do not hard-code these anywhere that gets committed

- **Email addresses** or personal names — use a placeholder like `you@example.com`.
- The **GCP project id** — use `<project-id>` / `your-project-id`.
- **Deployed Cloud Run URLs** (`https://<service>-<hash>-<region>.a.run.app`): the
  hash is derived from the project number, so it identifies the project. Use
  `<brain_url>`, `<auth_url>`, `<ui_url>` placeholders.
- **Service-account emails**, bucket names, or anything containing the project id.
- **Absolute local paths** or the machine username.

Real values live only where git ignores them:

- `config/*.tfvars` (project id, invoker identities, the Google client secret),
- the deployed bucket policy at `gs://<index_bucket>/policy.yaml` (real principals
  are granted there at runtime, not in the tracked `config/*.policy.yaml`, which
  stays `@example.com`),
- `ui/data/` (generated by the exporter).

When you need a real value at runtime (a token, a URL, a grant), fetch it
(`terraform -chdir=infra output`, `gcloud ...`) or upload it to the bucket. Do not
write it into a file that will be committed. Prefer placeholders in every example
and test.

## Before committing

Scan the staged diff for leaks: email addresses (an `@`), `a.run.app` URLs, and
your project id. For example:

```sh
git diff --cached | grep -inE '@[a-z0-9.-]+\.[a-z]{2,}|[a-z0-9-]+\.a\.run\.app'
```

If it matches anything that is a real address, URL, or id, replace it with a
placeholder before committing.
